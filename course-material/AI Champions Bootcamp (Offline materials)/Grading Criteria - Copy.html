Project Type 2 - Capstone Assignment: Building an Interactive LLM-Powered Solution

🎯 Objective

Develop a web-based application that enables citizens to interact seamlessly with publicly available information regarding a specific process or transaction with the government. The application should consolidate information or data from multiple official or trustworthy sources, facilitate a deeper understanding through interactive engagements, and present the relevant information in the way that is customised based user inputs and in the highly effective formats for the users to consume the information.

💡 If you opt for Project Type 1, you can skip this assignment (DO NOT make any submission in this category).
There will be a separate "assignment" page that is similar to this for teams who opt for Project Type 1.

✅ Assessment Criteria

Qualifying Criteria: A Working WebApp that can be accessed via Internet.

  1. Functionality & User Experience (35%):
    • The application meets all specified requirements and functions as intended.
    • Intuitive and user-friendly interface across multiple pages.
  2. Technical Implementation (25%):
    • Proper use of prompt engineering, prompt chaining techniques, and tools for LLM Agents.
    • Safeguard the app from prompt injection and minimise the chances of which the app being exploited.
  3. Innovation and Creativity (15%):
    • Unique features that enhance user interaction and understanding.
    • Creative solutions to data integration and presentation challenges.
  4. Documentation (25%):
    • Comprehensive and clear documentation of "About Us' and "Methodology" pages.

 

Note: A passing score is 60%. Only "Pass" or "Fail" will be shared.
Plagiarism is taken seriously. If any submission is found to be plagiarized, both the original and the copied work will receive a "Fail."
